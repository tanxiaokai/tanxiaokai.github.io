Rabbitmq使用

发表于 2017-02-24 | 分类于 开发 ， RabbitMq

一、下载及安装

首先去官网下载最新版本：[http://www.rabbitmq.com/download.html]

需要在CentOS上安装，所以下载对应的RPM包。安装过程就非常方便了，先安装一个erlang，直接在rabbitmq官网上下载到合适的RPM包，[http://www.rabbitmq.com/releases/erlang/]

rpm -ihv erlang-18.3.4.4-1.el7.centos.x86_64.rpm
rpm -ihv rabbitmq-server-3.6.6-1.el7.noarch.rpm

遇到以下错误，需要安装socat

警告：rabbitmq-server-3.6.6-1.el7.noarch.rpm: 头V4 RSA/SHA1 Signature, 密钥 ID 6026dfca: NOKEY
错误：依赖检测失败：
    socat 被 rabbitmq-server-3.6.6-1.el7.noarch 需要

安装方法

wget ftp://ftp.is.co.za/mirror/fedora.redhat.com/epel/6/x86_64/socat-1.7.2.3-1.el6.x86_64.rpm
rpm -ihv socat-1.7.2.3-1.el6.x86_64.rpm

二、服务启动关闭

CentOS7 启动

systemctl start rabbitmq-server 
systemctl enable rabbitmq-server #开机启动

CentOS6 启动

/etc/init.d/rabbitmq-server start    
或    
service rabbitmq-service start

三、启用Web后台

  1. 创建用户

     rabbitmqctl add_user admin admin123

  2. 查看用户

     rabbitmqctl list_users

  3. 设置用户组

     rabbitmqctl set_user_tags admin administrator

  4. 开启Web管理插件

     rabbitmq-plugins enable rabbitmq_management

  5. 查看端口

     netstat -lnutp
  6. 访问后台 [http://192.168.1.114:15672]

四、配置

  1. 配置文件

    Rabbitmq的配置文件有三个，位于/etc/rabbitmq/，
    这三个文件分别是：

    （1）enabled_plugins, 设置允许的插件列表，该配置文档的格式为erlang的列表格式如
    [rabbitmq_management,rabbitmq_visualiser].
    （2）rabbitmq.conf 设置rabbitmq的运行参数。如果是用rpm包安装，可从默认docs目录复制配置文件样例

     cp /usr/share/doc/rabbitmq-server-3.6.6/rabbitmq.config.example /etc/rabbitmq.config

    常用配置如下:

     tcp_listerners    #设置rabbimq的监听端口，默认为[5672]。
 disk_free_limit   #磁盘低水位线，若磁盘容量低于指定值则停止接收数据，默认值为{mem_relative, 1.0},即与内存相关联1：1，也可定制为多少byte.
 vm_memory_high_watermark  #设置内存低水位线，若低于该水位线，则开启流控机制，默认值是0.4，即内存总量的40%。
 hipe_compile     #将部分rabbimq代码用High Performance Erlang compiler编译，可提升性能，该参数是实验性，若出现erlang vm segfaults，应关掉。
 force_fine_statistics    #该参数属于rabbimq_management，若为true则进行精细化的统计，但会影响性能。
 frame_max     #包大小，若包小则低延迟，若包则高吞吐，默认是131072=128K。
 heartbeat     #客户端与服务端心跳间隔，设置为0则关闭心跳，默认是600秒。

    （3）rabbitmq-env.conf rabbitmq的环境参数配置 

     RABBITMQ_NODE_IP_ADDRESS= //IP地址，空串bind所有地址，指定地址bind指定网络接口
 RABBITMQ_NODE_PORT=       //TCP端口号，默认是5672
 RABBITMQ_NODENAME=        //节点名称。默认是rabbit
 RABBITMQ_CONFIG_FILE= //配置文件路径 ，即rabbitmq.config文件路径
 RABBITMQ_MNESIA_BASE=     //mnesia所在路径
 RABBITMQ_LOG_BASE=        //日志所在路径
 RABBITMQ_PLUGINS_DIR=     //插件所在路径

五、rabbitmqadmin使用

  1. 配置rabbitmqadmin

    #查找位置
find /var/lib/rabbitmq/ -name "rabbitmqadmin"

#复制，路径更加查找的结果来
cp -a /var/lib/rabbitmq/rabbit@*-plugins-expand/.../rabbitmqadmin /usr/bin/rabbitmqadmin   

#添加执行权限
chmod +x /usr/bin/rabbitmqadmin

    ​

  2. 使用示例

    #!/bin/bash
#该脚本用于初始化新用户的RabbitMq相关环境

#参数1是新用户对应的vHost名称
vHost=$1

#RabbitMq管理员用户名和密码
adminName=admin
password='123456'
topicName='ak.topic'

echo "初始化[${vHost}]数据"

#删除vHost
rabbitmqadmin -u $adminName -p $password delete vhost name=${vHost}

#创建vHost
rabbitmqadmin -u $adminName -p $password declare vhost name=${vHost}

#赋予权限
rabbitmqadmin -u $adminName -p $password declare permission vhost=${vHost} user=$adminName configure=".*" write=".*" read=".*"

#创建Exchanges
rabbitmqadmin -u $adminName -p $password  -V $vHost declare exchange name=$topicName type=topic

#创建queue
rabbitmqadmin -u $adminName -p $password -V $vHost declare queue name=batch_task durable=true
rabbitmqadmin -u $adminName -p $password -V $vHost declare queue name=eis_queue durable=true
rabbitmqadmin -u $adminName -p $password -V $vHost declare queue name=settlement_biz_queue durable=true
rabbitmqadmin -u $adminName -p $password -V $vHost declare queue name=spd_biz_queue durable=true

#创建队列绑定关系
rabbitmqadmin -u $adminName -p $password -V $vHost declare binding source=$topicName destination_type="queue" destination="batch_task" routing_key="create_batch_task"
rabbitmqadmin -u $adminName -p $password -V $vHost declare binding source=$topicName destination_type="queue" destination="eis_queue" routing_key="sync_task"
rabbitmqadmin -u $adminName -p $password -V $vHost declare binding source=$topicName destination_type="queue" destination="settlement_biz_queue" routing_key="spd_biz"
rabbitmqadmin -u $adminName -p $password -V $vHost declare binding source=$topicName destination_type="queue" destination="spd_biz_queue" routing_key="spd_biz"

echo "初始化[${vHost}]数据-完成"

#显示vhosts
echo "------------------------vhosts-------------------------------"
rabbitmqadmin -u $adminName -p $password -V $vHost list vhosts

#显示exchanges
echo "------------------------exchanges-------------------------------"
rabbitmqadmin -u $adminName -p $password -V $vHost list exchanges

#显示queues
echo "-------------------------queues---------------------------------"
rabbitmqadmin -u $adminName -p $password -V $vHost list queues

#显示bindings
echo "--------------------------bindings------------------------------"
rabbitmqadmin -u $adminName -p $password -V $vHost list bindings
