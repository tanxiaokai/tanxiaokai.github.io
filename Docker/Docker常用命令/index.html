Docker常用命令

发表于 2018-03-16 | 分类于 Docker

Docker常用命令

容器生命周期管理 — docker [run|start|stop|restart|kill|rm|pause|unpause]
容器操作运维 — docker [ps|inspect|top|attach|events|logs|wait|export|port]
容器rootfs命令 — docker [commit|cp|diff]
镜像仓库 — docker [login|pull|push|search]
本地镜像管理 — docker [images|rmi|tag|build|history|save|import]
其他命令 — docker [info|version]

安装docker (Centos7)

yum install docker

启动docker服务

systemctl start docker.service

设置docker服务开机启动

systemctl enable docker.service

关闭docker服务

systemctl stop docker.service

列出本地已经存在的镜像

Usage: docker images
示例:  
docker images

结果：
REPOSITORY   TAG      IMAGE ID            CREATED             SIZE
centos      latest    328edcd84f1b        4 weeks ago         193MB

在docker镜像中心中搜索镜像

Usage: docker search TERM
示例:  
docker search centos

结果：
NAME       DESCRIPTION                     STARS     OFFICIAL   AUTOMATED
centos     The official build of CentOS.   3608      [OK]
ansible/centos7-ansible  Ansible on Centos7   100    [OK]

从docker镜像中心下载镜像

Usage: docker pull [OPTIONS] NAME[:TAG]
示例:  
docker pull centos

通过镜像启动一个容器

Usage: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
示例1 在后台启动一个容器，并且暴露容器端口9000映射到Host的8081端口上:  
docker run -itd -p 8081:9000 centos
示例2 在后台启动一个容器，将宿主机的数据库目录/server/mysql-data挂载到容器/db上:
docker run -itd -p 3306:3306 -v /server/mysql-data:/db centos
示例3:
docker run -itd --name=server-db --hostname=docker-db1 -p 3306:3306 -v /server/mysql-data:/db centos 

参数详解
-d, --detach=false        指定容器运行于前台还是后台，默认为false     
-i, --interactive=false   打开STDIN，用于控制台交互    
-t, --tty=false           分配tty设备，该可以支持终端登录，默认为false    
-u, --user=""             指定容器的用户    
-a, --attach=[]           登录容器（必须是以docker run -d启动的容器）  
-w, --workdir=""          指定容器的工作目录   
-c, --cpu-shares=0        设置容器CPU权重，在CPU共享场景使用    
-e, --env=[]              指定环境变量，容器中可以使用该环境变量    
-m, --memory=""           指定容器的内存上限    
-P, --publish-all=false   指定容器暴露的端口    
-p, --publish=[]          指定容器暴露的端口   
-h, --hostname=""         指定容器的主机名    
-v, --volume=[]           给容器挂载存储卷，挂载到容器的某个目录    
--volumes-from=[]         给容器挂载其他容器上的卷，挂载到容器的某个目录  
--cap-add=[]              添加权限，权限清单详见：
                          http://linux.die.net/man/7/capabilities    
--cap-drop=[]             删除权限，权限清单详见：
                          http://linux.die.net/man/7/capabilities    
--cidfile=""              运行容器后，在指定文件中写入容器PID值，
                          一种典型的监控系统用法    
--cpuset=""               设置容器可以使用哪些CPU，此参数可以用来容器独占CPU    
--device=[]               添加主机设备给容器，相当于设备直通    
--dns=[]                  指定容器的dns服务器    
--dns-search=[]           指定容器的dns搜索域名，
                          写入到容器的/etc/resolv.conf文件    
--entrypoint=""           覆盖image的入口点    
--env-file=[]             指定环境变量文件，文件格式为每行一个环境变量    
--expose=[]               指定容器暴露的端口，即修改镜像的暴露端口    
--link=[]                 指定容器间的关联，使用其他容器的IP、env等信息    
--lxc-conf=[]             指定容器的配置文件，只有在指定--exec-driver=lxc时使用 
--name=""                 指定容器名字，后续可以通过名字进行容器管理，
                          links特性需要使用名字    
--net="bridge"            容器网络设置:  
                          bridge 使用docker daemon指定的网桥      
                          host    //容器使用主机的网络    
                          container:NAME_or_ID >
                          //使用其他容器的网路，共享IP和PORT等网络资源    
                          none 容器使用自己的网络（类似--net=bridge），
                          但是不进行配置   
--privileged=false        指定容器是否为特权容器，特权容器拥有所有的capabilities 
--restart="no"            指定容器停止后的重启策略: 
                            no：容器退出时不重启    
                            on-failure：容器故障退出（返回值非零）时重启   
                            always：容器退出时总是重启    
--rm=false                指定容器停止后自动删除容器
                          (不支持以docker run -d启动的容器)    
--sig-proxy=true          设置由代理接受并处理信号，
                          但是SIGCHLD、SIGSTOP和SIGKILL不能被代理

查看正在运行中的容器

Usage: docker ps
示例:  
docker ps

结果：
CONTAINER ID  IMAGE   COMMAND   CREATED STATUS  PORTS  NAMES
fcbba0473fea  centos  "/bin/bash"   5   seconds ago    Up 3 seconds

连接到正在运行中的容器

Usage: docker attach CONTAINER_ID
示例:  官方提供的方式 *注意：这种方式连接到容器，必须使用ctrl+p ctrl+q退出，如果使用ctrl+d 或者输入exit退出，容器将会被停止
docker attach fcbba0473fea

Usage: docker exec -it CONTAINER_ID /bin/bash 
示例:  用这种方式连接不会出现关闭容器的误操作
docker exec -it fcbba0473fea /bin/bash

正在运行中的容器中执行linux命令

Usage: docker exec [OPTIONS] CONTAINER_ID COMMAND [ARG...]
示例:
docker exec fcbba0473fea echo 'goodbye world'

停止正在运行的容器

Usage: docker stop CONTAINER_ID
示例:
docker stop fcbba0473fea

删除已经存在的容器

Usage: docker rm CONTAINER_ID
示例:
docker rm fcbba0473fea

###调整docker容器默认的空间

docker容器默认的空间是10G,如果想指定默认容器的大小，按照以下步骤进行

vi /etc/sysconfig/docker-storage 
DOCKER_STORAGE_OPTIONS="--storage-opt dm.basesize=20G"

systemctl stop docker.service
rm -rf /var/lib/docker
systemctl start docker.service

将容器转成镜像

Usage: docker commit -m "修改信息" -a "作者" CONTAINER_ID IMAGE_TAG_NAME
示例:
docker commit -m "add jdk" -a "tank" d316f2bd1357 spd_docker_has_oracle:0.1

通过Dockerfile文件构建镜像

Usage: docker build -t IMAGE_TAG_NAME PATH
示例:
docker build -t spd_docker:0.1 ./

docker build -f ./Dockerfile-centos7 -t spd_docker:0.2 ./

上传镜像到私服

1.首先登陆私服(admin aoka2017)
docker login 120.77.2.93:12345
2.输入nexus的账号和密码
3.设置tag
docker tag spd_docker:0.1 120.77.2.93:12345/spd_docker:0.1
4.push镜像
docker push 120.77.2.93:12345/spd_docker:0.1
5.下载镜像
docker pull 120.77.2.93:12345/spd_docker:0.1

###中间有些问题，因为私服不支持https请求，需要修改配置文件, 修改完后重启docker服务
###Mac下路径
vi ~/.docker/daemon.json (Mac)
vi /etc/docker/daemon.json (linux)
###添加代码  
{
  "insecure-registries" : [
    "120.77.2.93:12345"
  ]
}

删除镜像

docker rmi

更新配置

docker update 配置 服务名

示例： 
docker update -m 1536M test  #更新内存大小

从宿主机拷文件到容器里面

docker cp 要拷贝的文件路径 容器名：要拷贝到容器里面对应的路径

示例： 
docker cp test.java z2:/opt/
