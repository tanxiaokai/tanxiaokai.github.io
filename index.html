Rabbitmq使用

发表于 2017-02-24 | 分类于 开发 ， RabbitMq

一、下载及安装

首先去官网下载最新版本：[http://www.rabbitmq.com/download.html]

需要在CentOS上安装，所以下载对应的RPM包。安装过程就非常方便了，先安装一个erlang，直接在rabbitmq官网上下载到合适的RPM包，[http://www.rabbitmq.com/releases/erlang/]

rpm -ihv erlang-18.3.4.4-1.el7.centos.x86_64.rpm
rpm -ihv rabbitmq-server-3.6.6-1.el7.noarch.rpm

遇到以下错误，需要安装socat

警告：rabbitmq-server-3.6.6-1.el7.noarch.rpm: 头V4 RSA/SHA1 Signature, 密钥 ID 6026dfca: NOKEY
错误：依赖检测失败：
    socat 被 rabbitmq-server-3.6.6-1.el7.noarch 需要

安装方法

wget ftp://ftp.is.co.za/mirror/fedora.redhat.com/epel/6/x86_64/socat-1.7.2.3-1.el6.x86_64.rpm
rpm -ihv socat-1.7.2.3-1.el6.x86_64.rpm

Git SSH Key 生成步骤

发表于 2017-02-16 | 分类于 Git

一 、设置Git的user name和email

git config --global user.name "username"
git config --global user.email "email@gmail.com"

二、生成SSH密钥过程

1.查看是否已经有了ssh密钥, 如果没有密钥则不会有此文件夹，有则备份删除

cd ~/.ssh

2.生存密钥

$ ssh-keygen -t rsa -C "hostname"

按3个回车，密码为空。
最后得到了两个文件：id_rsa和id_rsa.pub

3.添加密钥到ssh

ssh-add 文件名
ssh-add ~/.ssh/id_rsa.pub

需要之前输入密码。

4.添加ssh密钥，这要添加的是“id_rsa.pub”里面的公钥。

cat ~/.ssh/id_rsa.pub

5.测试：

ssh git@192.168.1.114.com

Maven继承

为什么要使用继承

我们在项目开发的过程中，大多数情况都会把项目拆分成多个子模块独立开发。这些模块在各自的开发过程中都可能引入很多第三方依赖包，如Spring、Junit、JSON等。这时如何问题来了，如何保证所有子模块依赖的第三方包使用的版本一致？如何能快速对所有子模块对应依赖包的版本切换？Maven给我提供的项目继承关系可以很好的解决这个问题。
