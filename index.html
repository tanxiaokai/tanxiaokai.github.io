Gogs安装笔记

发表于 2018-02-03 | 分类于 开发 ， Git

​ Gogs是一款类似Github的开源文件／代码管理系统（基于Git）。它比GitLab小巧很多，占用的系统资源也少。比较适合个人、或者小团队搭建。不过感觉这个项目还比较新，还有一些小的问题。我在Centos 7安装时，遇到了些小麻烦。

一、下载安装包

  • 官网文档

  • 创建git用户

    adduser git

  • 二进制包 下载 

    su git
cd /home/git
wget https://dl.gogs.io/0.11.34/raspi2_armv6.zip  --树莓派
wget https://dl.gogs.io/0.11.34/linux_amd64.zip   --linux 64

  • 解压缩安装包

    unzip raspi2_armv6.zip    --树莓派
unzip linux_amd64.zip     --linux 64

二、启动、配置服务

  • 直接启动

    ./gogs web

  • 首次使用需要进行配置

    访问Web：http://localhost:3000

    1. 执行Mysql脚本创建数据库

      SET GLOBAL innodb_file_per_table = ON,
        innodb_file_format = Barracuda,
        innodb_large_prefix = ON;
DROP DATABASE IF EXISTS gogs;
CREATE DATABASE IF NOT EXISTS gogs CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
GRANT ALL PRIVILEGES ON gogs.* TO 'gogs'@'%' IDENTIFIED BY 'aoka20!*' WITH GRANT OPTION;

    2. 创建git仓库主目录

      mkdir /home/git/gogs-repositories

    3. 在页面填写对应内容

    4. 在点击安装后，可能会出现以下错误，需要不停的执行修复表的sql，执行后在点立即安装，直到所有的sql语句都执行成功为止：

      数据库设置不正确：Error 1709: Index column size too large. The maximum column size is 767 bytes.

      修复表sql

      use gogs;
alter table user ROW_FORMAT=DYNAMIC;
alter table repository ROW_FORMAT=DYNAMIC;
alter table login_source ROW_FORMAT=DYNAMIC;
alter table protect_branch ROW_FORMAT=DYNAMIC;
alter table protect_branch_whitelist ROW_FORMAT=DYNAMIC;
alter table email_address ROW_FORMAT=DYNAMIC;

  • 使用supervisor启动

    添加配置文件

     vi /etc/supervisor/conf.d/gogs.conf

    添加以下内容

    [program:gogs]
directory=/home/git/gogs/         
command=/home/git/gogs/gogs web 
autostart=true
autorestart=true
startsecs=10
stdout_logfile=/var/log/gogs/stdout.log
stdout_logfile_maxbytes=1MB
stdout_logfile_backups=10
stdout_capture_maxbytes=1MB
stderr_logfile=/var/log/gogs/stderr.log
stderr_logfile_maxbytes=1MB
stderr_logfile_backups=10
stderr_capture_maxbytes=1MB
user = git
environment = HOME="/home/git", USER="git"

    更新配置

    supervisorctl update
supervisorctl reload

三、 导入已有git仓库

  • 在gogs里面新建要导入的git仓库。注意一定需要同名。例如：导入yard_service

    # 在SPD组织下，新建 yard_service 仓库。
# 切换 git 用户
  su git
# 删除创建的目录
  rm -rf /home/git/gogs-repositories/spd/yard_service.git
# 拷贝原始git仓库
  cp -rf /home/git/repositories/yard_service.git  /home/git/gogs-repositories/spd/
# 重新初始化git
  cd  /home/git/gogs-repositories/spd/yard_service.git
  git init --bare

Rabbitmq使用

发表于 2017-02-24 | 分类于 开发 ， Java

一、下载及安装

首先去官网下载最新版本：[http://www.rabbitmq.com/download.html]

需要在CentOS上安装，所以下载对应的RPM包。安装过程就非常方便了，先安装一个erlang，直接在rabbitmq官网上下载到合适的RPM包，[http://www.rabbitmq.com/releases/erlang/]

rpm -ihv erlang-18.3.4.4-1.el7.centos.x86_64.rpm
rpm -ihv rabbitmq-server-3.6.6-1.el7.noarch.rpm

遇到以下错误，需要安装socat

警告：rabbitmq-server-3.6.6-1.el7.noarch.rpm: 头V4 RSA/SHA1 Signature, 密钥 ID 6026dfca: NOKEY
错误：依赖检测失败：
    socat 被 rabbitmq-server-3.6.6-1.el7.noarch 需要

安装方法

wget ftp://ftp.is.co.za/mirror/fedora.redhat.com/epel/6/x86_64/socat-1.7.2.3-1.el6.x86_64.rpm
rpm -ihv socat-1.7.2.3-1.el6.x86_64.rpm

二、服务启动关闭

CentOS7 启动

systemctl start rabbitmq-server

CentOS6 启动

/etc/init.d/rabbitmq-server start    
或    
service rabbitmq-service start

三、启用Web后台

  1. 创建用户

     rabbitmqctl add_user admin admin123

  2. 查看用户

     rabbitmqctl list_users

  3. 设置用户组

     rabbitmqctl set_user_tags admin administrator

  4. 开启Web管理插件

     rabbitmq-plugins enable rabbitmq_management

  5. 查看端口

     netstat -lnutp
  6. 访问后台 [http://192.168.1.114:15672]

四、配置

  1. 配置文件

    Rabbitmq的配置文件有三个，位于/etc/rabbitmq/，
    这三个文件分别是：

    （1）enabled_plugins, 设置允许的插件列表，该配置文档的格式为erlang的列表格式如
    [rabbitmq_management,rabbitmq_visualiser].
    （2）rabbitmq.conf 设置rabbitmq的运行参数。如果是用rpm包安装，可从默认docs目录复制配置文件样例

     cp /usr/share/doc/rabbitmq-server-3.6.6/rabbitmq.config.example /etc/rabbitmq.config

    常用配置如下:

     tcp_listerners    #设置rabbimq的监听端口，默认为[5672]。
 disk_free_limit   #磁盘低水位线，若磁盘容量低于指定值则停止接收数据，默认值为{mem_relative, 1.0},即与内存相关联1：1，也可定制为多少byte.
 vm_memory_high_watermark  #设置内存低水位线，若低于该水位线，则开启流控机制，默认值是0.4，即内存总量的40%。
 hipe_compile     #将部分rabbimq代码用High Performance Erlang compiler编译，可提升性能，该参数是实验性，若出现erlang vm segfaults，应关掉。
 force_fine_statistics    #该参数属于rabbimq_management，若为true则进行精细化的统计，但会影响性能。
 frame_max     #包大小，若包小则低延迟，若包则高吞吐，默认是131072=128K。
 heartbeat     #客户端与服务端心跳间隔，设置为0则关闭心跳，默认是600秒。

    （3）rabbitmq-env.conf rabbitmq的环境参数配置 

     RABBITMQ_NODE_IP_ADDRESS= //IP地址，空串bind所有地址，指定地址bind指定网络接口
 RABBITMQ_NODE_PORT=       //TCP端口号，默认是5672
 RABBITMQ_NODENAME=        //节点名称。默认是rabbit
 RABBITMQ_CONFIG_FILE= //配置文件路径 ，即rabbitmq.config文件路径
 RABBITMQ_MNESIA_BASE=     //mnesia所在路径
 RABBITMQ_LOG_BASE=        //日志所在路径
 RABBITMQ_PLUGINS_DIR=     //插件所在路径

如何返回方法的参数名称

发表于 2016-12-28 | 分类于 开发 ， Java

这两天接手一个新项目的代码，需要对整个项目的Spring框架从3.2.1 升级到4.2.9。因为涉及底层改动比较大，需要对整个服务进行全面的测试。我想做一个测试功能，把服务所有能请求的url，对两个不同版本的输出进行比对来验证升级是否没有问题。在反射MVC的所有Mapping的请求时，发现只能反射方法中的参数类型，而不能获取参数的名称。在网上查了些资料终于找到了解决的方法。

获取SpringMvc所有注册的请求信息

//核心包

import org.springframework.web.method.HandlerMethod;
import org.springframework.web.servlet.mvc.method.RequestMappingInfo;
import org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerMapping;

//关键代码
{
    Map<RequestMappingInfo, HandlerMethod> handlerMethods = requestMappingHandlerMapping.getHandlerMethods();
    Set<RequestMappingInfo> keySet = handlerMethods.keySet();

    for (RequestMappingInfo requestMappingInfo : keySet) {
            // 请求路径
            String path = requestMappingInfo.getPatternsCondition().toString();

            // 请求方法
            String requestMethod = requestMappingInfo.getMethodsCondition().toString();

            // 返回header类型
            String responseType = requestMappingInfo.getProducesCondition().toString();

            // 获取参数名称
            HandlerMethod method = handlerMethods.get(requestMappingInfo);
            List<String> parameters = parseParamNames(method.getBeanType(), method.getMethod().getName());
    }
}

解析方法的参数名称

//需要依赖工具类 
<dependency>
    <groupId>javassist</groupId>
    <artifactId>javassist</artifactId>
    <version>3.4.GA</version>
</dependency>

//关键代码
private List<String> parseParamNames(Class clazz, String methodName ){
        List<String> returnList = new ArrayList();
        try { 
            //官方鼓励使用这种方法
            ClassPool pool = ClassPool.getDefault();
            ClassClassPath classPath = new ClassClassPath(clazz);
            pool.insertClassPath(classPath);

            //使用绝对路径方式
            //ClassPool pool = ClassPool.getDefault();
            //pool.insertClassPath("/path/xxxxxxxxxxx.jar");

            CtClass cc = pool.get(clazz.getName());  
            CtMethod cm = cc.getDeclaredMethod(methodName);  

            //使用javaassist的反射方法获取方法的参数名  
            MethodInfo methodInfo = cm.getMethodInfo();  
            CodeAttribute codeAttribute = methodInfo.getCodeAttribute();  
            LocalVariableAttribute attr = (LocalVariableAttribute) codeAttribute.getAttribute(LocalVariableAttribute.tag);  
            if (attr == null) {
                return returnList;
            }  
            String[] paramNames = new String[cm.getParameterTypes().length];  
            int pos = Modifier.isStatic(cm.getModifiers()) ? 0 : 1;  
            for (int i = 0; i < paramNames.length; i++)  {
                paramNames[i] = attr.variableName(i + pos);
            }
            for (int i = 0; i < paramNames.length; i++) {  
                returnList.add(paramNames[i]);  
            }
        }  catch(Exception e){
            e.printStackTrace();
        }
        return returnList;
}

在做这个解析的时候遇到了javassist.NotFoundException 异常，主要原因是我使用的jar包的方式运行，classpath没有找到。按照我代码中的两种方式可以解决这种问题。
